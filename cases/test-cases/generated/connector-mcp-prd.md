## Method 1: Equivalence Partitioning

### MCP 面板显示等价类

| 输入条件 | 有效等价类 | ID | 无效等价类 | ID |
|---------|----------|-----|----------|-----|
| 工具执行状态 | 执行中（实时更新） | V1 | — | — |
| | 执行完成（结果保持） | V2 | — | — |
| 执行结果格式 | 有效 JSON 结果 | V3 | 空结果 | I1 |

**TC-PRD-MCP-001**: MCP 工具执行时自动打开右侧 Workspace Panel
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** Agent 执行 MCP 工具（如 voice_get_scenario_variables）
- **操作步骤:** 1. 触发 Agent 执行 MCP 工具 2. 观察右侧面板
- **预期结果:** 右侧 Workspace Panel 自动打开，标题显示工具名称

**TC-PRD-MCP-002**: MCP 面板显示 JSON 格式执行结果
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** MCP 工具执行完成
- **操作步骤:** 1. 查看 Result 区域内容
- **预期结果:** Result 区域显示 JSON 格式的执行结果，格式化且可读

## Method 4: State Transition Testing

### MCP 面板状态

状态：S0=关闭, S1=执行中（实时更新）, S2=执行完成（结果保持）

| 当前状态 | 事件 | 下一状态 |
|---------|------|---------|
| 关闭 | 工具开始执行 | 执行中 |
| 执行中 | 工具执行完成 | 执行完成 |
| 执行完成 | 用户关闭面板 | 关闭 |

**TC-PRD-MCP-003**: 面板从关闭到执行中 — 自动打开并实时更新
- **优先级:** P0
- **测试类型:** 状态迁移
- **前置条件:** 右侧 Workspace Panel 处于关闭状态
- **操作步骤:** 1. 触发 MCP 工具执行
- **预期结果:** 面板自动打开，显示 "Real time" 指示器，结果区域实时更新

**TC-PRD-MCP-004**: 面板从执行中到完成 — 结果保持可查看
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** MCP 工具正在执行中
- **操作步骤:** 1. 等待工具执行完成
- **预期结果:** "Real time" 指示器消失或变为完成状态，JSON 结果保持可查看

## Method 5: Scenario Method

**TC-PRD-MCP-005**: 完整 MCP 工具执行流程
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 用户在对话页面，Agent 可用
- **操作步骤:** 1. 发送触发 MCP 工具的消息 2. 等待面板自动打开 3. 观察工具名称标题 4. 观察实时执行结果 5. 等待执行完成 6. 查看最终 JSON 结果
- **预期结果:** 面板自动打开，标题为工具名称，实时显示执行过程，完成后 JSON 结果保持可查看

---

## Method 6: Error Guessing

**TC-PRD-MCP-006**: MCP 工具执行失败时面板显示错误信息
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** MCP 工具执行遇到错误
- **操作步骤:** 1. 触发一个会失败的 MCP 工具调用 2. 观察面板内容
- **预期结果:** 面板显示错误信息，不会白屏或卡在加载状态

**TC-PRD-MCP-007**: 连续执行多个 MCP 工具时面板内容正确切换
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 用户在对话页面
- **操作步骤:** 1. 触发第一个 MCP 工具执行 2. 立即触发第二个 MCP 工具执行 3. 观察面板标题和内容切换
- **预期结果:** 面板标题和内容正确切换到最新执行的工具，不会混淆显示
