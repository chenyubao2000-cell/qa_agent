Core Concept

核心理念

真实流量是最好的测试数据源。录制一次，回放多次，Agent 负责赋予语义。

流量录制解决"测试数据从哪来"的问题，
Agent 负责将录制的流量转化为有意义的测试用例和断言。

传统测试需要手动构造请求和预期响应，TestMirror 直接从真实生产/预发环境捕获完整的 request/response 对，经脱敏后作为测试输入。Agent 分析流量语义，自动生成 schema 校验、语义比对、性能回归等多维度断言，让每一条真实交互都成为守护质量的哨兵。

Mock Strategy

黑盒 Mock 策略

录制即 Mock -- 真实响应直接作为 Mock 数据源

📜

录制一次，回放多次

录制的真实响应直接作为 mock 数据源。无需手动编写 fixture，无需猜测 API 行为。真实流量自带完整的请求上下文和响应格式，是最可靠的 mock 来源。

Record Once, Replay Forever
🤖

对 LLM 调用尤其有效

LLM 响应非确定性，传统 mock 难以模拟。录制真实的 LLM 请求/响应对，在回放时提供稳定的基线，使 LLM 应用的回归测试成为可能。配合 LLM-as-Judge 评估语义一致性。

LLM-Native Testing
Mira Special Value

对 Mira 的特殊价值

已有基础设施天然契合 TestMirror 架构

📈

Langfuse LLM Trace

Mira 已部署 Langfuse，拥有完整的 LLM 调用 trace 数据。这些 trace 可以直接作为录制数据源，无需额外部署录制基础设施。每一条 trace 都包含完整的 prompt、completion、token 用量和延迟信息。

零成本冷启动
🌐

WebSocket Streaming 录制

Mira 核心交互基于 WebSocket streaming。TestMirror 的流量录制可覆盖完整的 WebSocket 会话，包括 streaming token、中断重连、并发消息等场景，确保核心交互路径的测试覆盖。

覆盖核心交互
Evaluation

方案评估

适用于已有生产流量的成熟 AI 应用

✅ 优势

  • 测试数据最真实，来自真实用户行为
  • 对 LLM 应用特别友好，天然解决非确定性问题
  • Agent 只做高价值工作（语义分析、断言生成）

⚠️ 劣势

  • 隐私合规风险，需严格的脱敏机制
  • 无法覆盖未发生过的场景（新功能、边界条件）
  • 录制基础设施部署和维护成本

🎯 适用场景

  • 已有生产流量的成熟 AI 应用
  • LLM 密集型产品的回归测试
  • 已部署可观测性平台（如 Langfuse）的团队
