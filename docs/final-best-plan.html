五、黑盒Mock策略（三层）

L1 — 网络层 Mock（覆盖80%场景）

工具：MSW (Mock Service Worker) + 现有 mock-server (Hono+Bun)

方式：Agent分析代码中的fetch/axios调用点 → 自动生成MSW handler → CI中自动注入

零侵入：不修改业务代码，在网络层拦截

L2 — LLM层 Mock（AI应用专用）

工具：Vercel AI SDK MockLanguageModelV2 + Langfuse录制回放

单元/集成测试：MockLanguageModelV2 提供确定性LLM响应（零成本）

E2E测试：从Langfuse trace导出真实LLM响应 → 序列化为fixture JSONL → 回放时零API调用

Eval：必须调用真实LLM（这是eval的价值所在）

L3 — 数据层 Mock

工具：Drizzle Schema → seed脚本 + fixture管理

方式：从Drizzle Schema自动生成seed数据 → worker-scope fixture隔离 → 测试前seed/测试后清理

复用：qa_agent的test-data-setup skill已实现3级回退（env→缓存→UI创建）

# Unit Test Agent

你是单元测试增量生成专家。

## 输入
- PR diff（由test-flow.yml提供）
- 项目的测试规范文件（unit-testing.md）
- 现有测试文件（作为风格参考）

## 工作流程

### Phase 1: 分析
1. 读取PR diff，提取新增/修改的函数列表
2. 读取项目测试规范（如果存在unit-testing.md则严格遵循）
3. 扫描已有测试文件，学习命名规范、断言风格、mock模式

### Phase 2: 生成
对每个需要测试的函数：
1. 分析函数签名、参数类型、返回类型、分支路径
2. 按等价类划分设计测试用例
3. 生成测试代码，遵循已有风格

### Phase 3: 输出
- 将测试建议格式化为PR comment（markdown代码块）
- 标注每个测试的覆盖目标（哪个函数、哪个分支）
- 如果函数<20行且无外部依赖，降级为sonnet模型生成

## 约束
- 不修改业务代码，只生成测试文件
- 不生成已有测试覆盖的函数的测试
- Mock外部依赖，不mock内部函数
- 遵循项目已有的断言库（expect/assert）

# Sentinel Agent

你是全时在线的质量守卫。

## 监控清单

### Sentry（每5分钟）
1. 调用Sentry API查询最近5分钟的新issue
2. 对每个新issue：分析stack trace → 定位源码函数 → 检查是否有对应测试
3. 如果无测试覆盖：生成回归测试建议 → 创建GitHub Issue或PR

### Langfuse（每小时）
1. 查询最近1小时的LLM trace统计
2. 计算关键指标：平均评分、错误率、延迟P95、token成本
3. 与历史基线对比，退化>10%时告警

### Railway（部署事件）
1. 收到部署webhook后等待30秒warmup
2. 执行smoke test suite（核心路径健康检查）
3. 通过→通知；失败→建议回滚+详细诊断

### DB（migration触发）
1. 检测Drizzle schema变更
2. 生成schema前后兼容性测试
3. 执行数据一致性验证

## 成本控制
- 日常轮询用haiku（低成本）
- 发现异常后升级到sonnet深入分析
- 每日API调用预算上限：$5

# Eval Agent

你是LLM质量评估专家。

## 工作模式

### 模式1: 构建eval dataset
1. 从Langfuse API导出最近N条trace
2. 提取input/output/tool_calls三元组
3. 按rubric维度标注（质量/安全/准确性/格式/tool调用正确性）
4. 输出为JSONL格式，存入eval-datasets/目录

### 模式2: 执行eval
1. 加载eval dataset
2. 对每条case执行LLM-as-Judge评分（使用Claude Batch API降低成本50%）
3. 生成评分报告：分维度得分+趋势对比+退化检测
4. 分数低于阈值时触发告警

### 模式3: Prompt回归
1. 检测prompt文件变更（git diff）
2. 自动触发eval对比：变更前baseline vs 变更后
3. 输出影响评估报告

## 评分维度
- accuracy: 回答准确性（0-5）
- safety: 安全合规（0-5）
- format: 格式符合要求（0-5）
- tool_use: tool调用正确性（0-5）
- latency: 响应延迟是否达标（pass/fail）

九、CI Workflow示例

test-flow.yml — PR级增量测试
name: TestFlow - Incremental Test Analysis
on:
  pull_request:
    types: [opened, synchronize]

jobs:
  analyze:
    name: Diff Impact Analysis
    runs-on: ubuntu-latest
    outputs:
      affected_packages: ${{ steps.affected.outputs.packages }}
      has_coverage_gaps: ${{ steps.gaps.outputs.has_gaps }}
      gap_files: ${{ steps.gaps.outputs.files }}
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: oven-sh/setup-bun@v2

      - name: Install dependencies
        run: bun install --frozen-lockfile

      - name: Affected Package Analysis
        id: affected
        run: |
          AFFECTED=$(bunx turbo run test --filter='...[origin/main...HEAD]' --dry-run=json | jq -r '.packages | join(",")')
          echo "packages=$AFFECTED" >> $GITHUB_OUTPUT

      - name: Coverage Gap Detection
        id: gaps
        run: |
          # 对比diff文件和coverage，找出缺测试的新代码
          bun run scripts/coverage-gap-detector.ts \
            --base origin/main \
            --head HEAD \
            --threshold 80 \
            --output gaps.json
          HAS_GAPS=$(jq '.gaps | length > 0' gaps.json)
          echo "has_gaps=$HAS_GAPS" >> $GITHUB_OUTPUT
          echo "files=$(jq -c '.gaps' gaps.json)" >> $GITHUB_OUTPUT

  suggest:
    name: AI Test Suggestions
    needs: analyze
    if: needs.analyze.outputs.has_coverage_gaps == 'true'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Generate Test Suggestions
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
        run: |
          # Claude API 分析diff + coverage gaps → 生成测试建议
          bun run scripts/ai-test-suggest.ts \
            --gaps '${{ needs.analyze.outputs.gap_files }}' \
            --style "$(cat unit-testing.md 2>/dev/null || echo 'default')" \
            --output suggestions.md

      - name: Post PR Comment
        uses: actions/github-script@v7
        with:
          script: |
            const fs = require('fs');
            const suggestions = fs.readFileSync('suggestions.md', 'utf8');
            await github.rest.issues.createComment({
              owner: context.repo.owner,
              repo: context.repo.repo,
              issue_number: context.issue.number,
              body: `## 🤖 AI Test Suggestions\n\n${suggestions}\n\n> 审核后回复 \`/approve-tests\` 自动提交`
            });

  selective-test:
    name: Run Affected Tests
    needs: analyze
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: oven-sh/setup-bun@v2
      - run: bun install --frozen-lockfile
