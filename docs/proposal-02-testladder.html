PROPOSAL 02

分层渐进式平台 TestLadder

不一步到位，按 ROI 从高到低分层建设。每层有明确的人 / Agent / 自动化边界，独立可用，逐步攀升。

核心理念

拒绝"大爆炸"式上线。按投资回报率（ROI）从高到低分三层递进建设， 每一层都有明确的人 / Agent / 自动化职责边界。 Layer 1 几乎零成本即可落地，团队在看到价值后再决定是否继续向上攀登。

渐进架构
三层阶梯

每一层独立可用，向上兼容。跌落风险最低，价值逐层累积。

Layer 1 — 立即落地 1 - 2 周

增强现有 CI + 单元测试 Agent

在现有 CI 流水线上叠加影响分析与覆盖缺口检测，引入 Agent 自动生成单元测试增量建议。 无需新基础设施，直接复用 Turborepo affected + import graph，几乎零额外成本。

Turborepo affected coverage diff unit-testing.md PR comment
Layer 2 — 中期建设 1 - 2 月

集成 qa_agent E2E + Mock 平台

引入 qa_agent 全流程 E2E 测试能力与 git-watcher 自动触发， 同时搭建 Mock 平台，Agent 分析新增外部依赖并自动生成 Mock 配置，实现端到端测试闭环。

qa_agent E2E git-watcher Mock Server 外部依赖分析
Layer 3 — 长期演进 3 - 6 月

性能测试 + 监控闭环 + 全自动

构建性能基准管理（k6 + SLA）、Sentry 回归测试自动化、Langfuse LLM 质量监控， 形成从检测到修复到评估的全自动闭环。Agent 占比达到 60%。

方案评估
优劣势与适用场景

+ 优势

  • 风险最低，每层独立可用，随时可以停下
  • Layer 1 几乎零成本，复用现有 CI 即可落地
  • 团队按自己的节奏逐步采纳，无需一次性投入
  • 价值可度量：每层上线后即可看到 ROI

− 劣势

  • 全面覆盖需要较长时间（3-6 个月达到 L3）
  • 层间集成可能存在缝隙，需要额外胶水代码
  • 如果团队在 L1 后停滞，可能错失 E2E 和监控闭环的价值

• 适用场景

  • 务实的团队，希望快速看到价值再决定是否继续投入
  • 资源有限，无法一次性投入大量人力和基础设施
  • 需要向管理层逐步证明 QA 自动化的 ROI
