--scope L1/L2/all
--exec/--no-exec
--report-linear

P3 执行测试（可选）
CMD --exec ?（默认 true）
A: --exec（默认）
VITEST 执行测试套件
输入
tests/mcp/{slug}/*.test.ts
使用 pool: "threads" +
singleThread: true
输出
results.json（vitest JSON 报告）
失败不中断，跑完全部用例；
quota 用尽自动 ctx.skip()
B: --no-exec
跳过执行
仅生成 spec + Excel
P4 报告
SONNET 解析 results.json
输入
results.json
schema-diff.report.md
输出
统计：total / passed / failed / skipped
按 P0/P1 分级 + 失败 detail
CMD 生成 {slug}-summary.md
输出
概览: 总数 / P0 通过率 / P1 通过率 / Skip
Schema Diff: PRD vs Server 摘要
失败分析: 用例名 / 错误 / 可能原因
质量门禁:
  • 协议合规 (L1): P0 100%
  • Schema 一致性: critical / warn 计数
  • Annotations: P0 100%
  • 正常路径 (L2): ≥ 95%
  • 边界 + 负面 (L2): ≥ 80%
  • 幂等性: 100%
