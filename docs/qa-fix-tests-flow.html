P0 初始化

输出

upgradeI18n, strategy

spec 头部元数据校验

执行 upgrade-i18n.md

升级完成
B: --skip-baseline
跳过 P1 基线执行
所有 spec 视为需修复
直接进入 P2
上游命令调用时默认此模式
C: 正常模式
进入 P1
先执行基线再修复
P1 筛选 + 执行基线（仅正常模式，--skip-baseline 跳过此区域）
CMD 查找非跳过 spec

输出
specFilesToExecute[]

无

全通过
无需修复

有失败
failedTests[]
继续 P2
P2 修复失败文件
USER 修复策略
上游调用(--skip-baseline)默认并行；交互模式由用户选择
Parallel 并行
SONNETCDP fix 子代理 × N（同时启动）
每个独立 CDP 探查
无跨文件上下文共享
快速但 token 多
Serial 串行
SONNETCDP fix 子代理 × N（逐个执行）
前一个的 cdpFindings
传给下一个 previousFixContext
慢但 token 省
每个 fix 子代理的执行流程
CDP 探查真实页面 → 分析错误 → 分类（测试问题 / 应用 bug）
→ 修复 spec/POM 定位器和断言 → 单文件验证
输出
{ status, fixes, cdpFindings, bugCount, changedAssertions }
P2.5 Handoff 同步 + Bug 汇总
CMD handoff-sync
对 assertionsChanged=true 的结果
更新对应 handoff JSON 中的断言字段
CMD 检测到应用 Bug？
A: 有 Bug
CMD 报告用户
列出检测到的 Bug
不上报 Linear
提示跑 /qa-run 正式上报
B: 无 Bug
跳过 Bug 汇总
P3 回归（仅修改过的文件）

输出
回归报告

CMD 生成修复报告
修复前/后对比 + 修复列表
Bug 列表 + 需人工审查项
流水线结束
