🎯 核心理念

LLM 输出天然不确定，传统断言（assertEqual / regex match）在 AI 应用中完全失效。TestEval 采用全新范式来解决这一根本矛盾。

🤖

LLM-as-Judge

用 Claude 等大模型作为评判者，对 LLM 输出进行多维度打分。支持自定义 Rubric，覆盖质量、安全、准确性、格式等维度。

📊

Golden Dataset 基准

人工标注初始数据集，结合 Langfuse trace 自动扩充。版本化管理，确保评估结果可复现、可对比。

🔄

回归检测

每次 Prompt 变更或模型切换，自动触发 Eval 对比。分数趋势可视化，退化即告警，保护 AI 行为质量。

🔗

Langfuse 天然集成

从 Langfuse trace 自动导出数据构建评估集，成本追踪、Token 分析一体化，形成可观测的 AI 质量闭环。

⚡ Agent 专用 Eval Skills

四个专用 Skill 构成 Agent 的评估能力核心。

📦
eval-dataset-builder
从 Langfuse trace 自动提取高质量样本，构建和扩充 Golden Dataset。支持版本化管理与去重。
⚖
llm-judge
基于 Claude 的多维度评分引擎。支持 Custom Rubric、Semantic Similarity、Regex/Keyword 等多种评判策略。
🔧
tool-use-validator
验证 LLM Tool 调用的参数正确性、调用顺序合理性。确保 Agent 的工具使用行为符合预期规范。
💰
cost-analyzer
分析 Token 使用模式、推理成本趋势，结合 Langfuse cost tracking 给出优化建议。
✅ 方案评估

✅ 优势

  • 唯一能真正测试 LLM 行为的方案，解决不确定性输出的质量保障难题
  • 与 Langfuse 天然集成，数据采集-评估-监控形成闭环
  • 多维度评分体系，覆盖质量、安全、准确性、格式等方面
  • 支持 Prompt 回归和模型切换评估，保护 AI 迭代质量

⚠ 劣势

  • 只覆盖 AI 行为，不覆盖常规代码逻辑的测试
  • LLM-as-Judge 本身存在不确定性，评分可能有波动
  • 需要人工标注初始 Golden Dataset，冷启动成本较高

🎯 适用场景

  • AI/LLM 应用的质量保障 -- 必备方案
  • 与传统单元测试、E2E 测试等方案互补，共同构成完整质量体系
  • Prompt 工程迭代频繁的团队尤其受益
  • 多模型切换 / A/B 实验场景
