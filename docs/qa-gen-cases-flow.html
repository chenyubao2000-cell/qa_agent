/qa-gen-cases 流程图

/qa-gen-cases [prd-path] [--output <dir>]

CMD 命令层 SONNET 生成
P0 初始化
CMD 读取配置
输入
.env
输出
QA_WORKSPACE_DIR, SOURCE_PROJECT_DIR, APP_LANGUAGES
CMD 确定输出目录
输入
--output / QA_WORKSPACE_DIR / cwd
输出
OUTPUT_DIR
仅创建 test-cases/ 目录
不初始化 playwright 基础设施
P1 读取需求文档
CMD 文档格式？
A: .md 格式
直接 Read()
B: .docx 格式
python-docx 转换
→ _prd-converted.md
CMD PRD 模块拆分
输入
文档内容
输出
按 ## 标题拆分的模块列表
P1.5 PRD 变更检测
CMD 计算 PRD Hash + 对比
输入
现有 *-prd.md 中的 PRD-hash + 当前模块 sha256
输出
每个模块的 prdChangeMode
CMD 模块变更状态
none
哈希匹配
跳过，已有用例最新
new
无现有 .md
从头生成
updated
哈希不匹配
增量更新
removed
模块已删除
标记废弃
P2 生成用例（仅 new / updated）
SONNET e2e-orchestrator
输入
source:"prd", prdFiles, prdModuleScope, prdChangeMode, projectContext
输出
{slug}-prd.md (含 PRD-hash 头部)
playwright-handoff-{slug}.json
仅生成 .md + handoff JSON，不生成 .test.ts / .page.ts，不执行测试
P2.5 验证门 V1 + V2
CMD V1: .md 文件验证
含 "## Merged Test Case List"
含至少 1 个 "**TC-" 条目
失败 → 报错停止
CMD V2: handoff JSON 验证
有效 JSON 数组
条目数 == TC 数（1:1 映射）
失败 → 报错停止
P3 Excel 导出
CMD Excel 导出脚本
输入
test-cases/generated/
输出
test-cases/excel/{slug}-prd.xlsx
CMD Excel 文件生成成功？
A: 成功
B: 失败 → 重试一次
CMD 重试成功？
成功
仍失败
报错
流水线结束
