P2 匹配已有 Spec → 策略
CMD 构建 slug → spec 映射
输入
Glob tests/e2e/testcases/generated/*.test.ts + pages/*.page.ts
输出
specMap: { slug → specPath }, pomMap
示例：task-cdp.test.ts → "task" / task-sidebar-cdp.test.ts → "task-sidebar"
CMD 提取模块关键词 + 匹配 specMap
① 从 changelist 提取关键词（3 种启发式）：
路由段：src/app/(dashboard)/task/[id]/page.tsx → "task"
组件名：src/components/TaskSidebar.tsx → PascalCase→kebab → "task-sidebar"
API 模块：src/api/canvas.ts → "canvas"
跳过：package.json, .d.ts, node_modules/ 等非 UI 文件
② 合并 Issue 派生关键词：
allModuleKeywords = deduplicate(changelistKeywords + issueModuleKeywords)
③ 用 contains / starts-with 匹配 specMap：
"task" → 匹配：task-cdp.test.ts, task-sidebar-cdp.test.ts, ...
"people-search" → 无匹配 → 归入 unmatchedModules
输出
matchedSpecs[] — 已有用例路径
unmatchedModules[] — 无对应 spec 的关键词
