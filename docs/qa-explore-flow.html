CMD 命令层 CDP 页面探查 SONNET 生成
P0 初始化
CMD 读取配置
输入
.env, CLAUDE.md
输出
projectContext (baseURL, authSetup,
appLanguages, techStack)
CMD 初始化工作区
phase-0-init.md 2a-2f
i18n + Auth 基础设施
CMD 确定探查 URL
用户参数 > PREVIEW_URL
输出
pageUrl
P1 初始扫描 + 识别功能区域
CDP 登录墙检测
A: 需要登录
CDP 自动登录
三层扫描发现选择器
CDP 填写凭证 → 登录
生成 auth.setup.ts
生成 sign-in POM
B: 不需要登录
直接扫描
CDP State_0 扫描
输入
pageUrl (登录后)
输出
State_0 DOM 结构 + a11y 树 + 截图
CMD 识别功能区域
输入
State_0 DOM 快照
识别信号：<form>→表单, [role="tab"]→选项卡,
[role="navigation"]→导航, [aria-haspopup]→下拉,
button "new/create"→弹窗, 独立内容块→内容区
输出
areas[] 按测试价值排序: form > tab > modal > nav > dropdown > content
CMD 目标区域过滤
A: 指定 targetArea
仅保留匹配区域
B: 未指定
保留所有区域
P2a 逐区域 CDP 探查（串行循环）
CDP 探查子代理 × 每个区域（串行）
输入
area, pageUrl, baselineFile, sourceProjectDir
Phase 0 源码预读 → Phase 3 BFS 交互探查
点击、悬停、填写、导航 → 记录状态转换
输出
新状态 + 新边 + discoveredAreas[] + coverageReport
CMD 动态发现新区域？
有 → 弹窗/选项卡面板/菜单/懒加载
追加到 areas 队列继续循环
无 → 继续下一区域
CMD maxAreas 达上限？
是 → 退出循环
否 → 继续
P2a.5 跨区域流发现
CMD 跨区域流存在？
检测 sourceArea 变化的边、URL 变化的边
A: 有
CDP 导航目标页
仅 1 跳 + State_0 扫描
记录 crossAreaFlows[]
B: 无
跳过
P2b 并行用例生成
SONNET e2e-orchestrator × N（每区域一个，并行）
输入
source:"cdp", baselineFile, areaScope, projectContext
输出
{slug}-{area-id}-cdp.md, handoff.json, *.xlsx
{slug}.page.ts, {slug}-{area-id}-cdp.test.ts
若有 crossAreaFlows → 额外启动一个 orchestrator 生成集成用例
CMD 验证门 V1-V5
通过 → 收集产出物
失败 → 删除该区域不完整产出物
CMD Excel 导出
{slug}-all-cases.xlsx
P3 交给 qa-fix-tests
CMD allSpecs 是否为空？
A: 空（已有全覆盖）
结束
B: 非空
CMD 执行 /qa-fix-tests --skip-baseline
CDP 校验定位器 → 修复 → 执行 → 回归
流水线结束
