/qa-run 流程图

/qa-run [--suite <type>] [--lang <lang>] [--slug <keyword>] [--source-filter <source>]

CMD 命令层 SONNET 执行/分析 LINEAR
P0 初始化 + 参数解析
CMD 读取配置
输入
.env
输出
projectContext, LINEAR_* 配置
CMD 验证基础设施
playwright.config.ts
fixtures.ts
*.test.ts 文件存在性
CMD spec 文件是否存在？
A: 有 spec 文件
B: 无 spec 文件
报错终止
请先运行 /qa-explore 或 /qa-run-prd
CMD 解析参数（4 个维度）
套件选择
--suite smoke / regression / full(默认) / P0-P2
"冒烟"→smoke, "回归"→regression
语言选择
--lang en / zh / all(默认)
"中文"→zh, "英文"→en
文件范围
指定路径 / --slug / --source-filter / 全部
"只跑登录"→slug=login
Headless
git-watcher 触发→true
其他→false
P1 执行测试
SONNET test-executor
输入
mode:"full", suite, specFiles, langProject, appLanguages
输出
playwright-results.json
playwright-report/index.html
P2 分析报告
SONNET report-analyzer
输入
playwright-results.json
changeSummary (可选, git-watcher)
relatedIssueKeys (可选)
输出
失败列表 + 路由策略
summary.md
打开 HTML 报告（非 headless）
P3 Linear 上报（条件）
CMD 是否有失败？
A: 有失败
SONNET bug-reporter
输入
failures[], linearTeamId
linearProjectId
输出
创建/追加 Linear Issue
B: 全通过
跳过 bug-reporter
无需 Linear 上报
CMD 更新 summary.md
注入 Linear Issue URL 到报告中
流水线结束

产出物

tests/reports/playwright-results.json始终
playwright-report/index.html始终
tests/reports/combined/summary.md始终
Linear Issues有失败时
