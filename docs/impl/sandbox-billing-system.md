● 下面是完整的沙箱计费体系梳理：    

  ---
  一、涉及的数据库表总览

  沙箱计费链路涉及 6 张表，数据流向为：

  sandbox_run → mira_usage + mira_usage_meter + mira_credit → mira_credit_ledger
                                                      ↑
                                                credit_rate (汇率查询)

  另外 mira_chats.sandboxId 存储任务与沙箱的关联。

  ---
  二、各表字段详解

  表 1：sandbox_run — 沙箱运行区间记录

  每次 resume→pause 产生一条记录。

  ┌────────────────┬────────────────┬─────────────────────────────────────────────────────────────────┐
  │      字段      │      类型      │                              说明                               │
  ├────────────────┼────────────────┼─────────────────────────────────────────────────────────────────┤
  │ id             │ varchar(36) PK │ 运行记录 ID（UUID），也作为 mira_usage.vendorRequestId 的幂等键 │
  ├────────────────┼────────────────┼─────────────────────────────────────────────────────────────────┤

  ├────────────────┼────────────────┼─────────────────────────────────────────────────────────────────┤
  │ messageId      │ varchar(36)    │ 关联消息 ID（哪轮对话触发的）                                   │
  ├────────────────┼────────────────┼─────────────────────────────────────────────────────────────────┤
  │ sandboxId      │ varchar(64)    │ E2B 沙箱实例 ID                                                 │
  ├────────────────┼────────────────┼─────────────────────────────────────────────────────────────────┤

  ├────────────────┼────────────────┼─────────────────────────────────────────────────────────────────┤

  ---
  三、完整计费公式链

  ① 运行时长
     runningSeconds = Math.ceil((pauseAt - resumeAt) / 1000)

  ② USD 费用
     costUsd = runningSeconds × $0.000046

  ③ 原始积分（未取整）
     rawCreditAmount = costUsd × creditsPerUnit   (线性公式)

  ④ 消息级汇总（该消息所有 type 的 credit 加总）
     sumRaw = Σ rawCreditAmount  (LLM + tool + sandbox)

  ⑤ 带进位的取整
     total = sumRaw + carryBefore  (上一条 ledger 的零头)
     amount = -floor(total)        (扣减的整数积分)
     carryAfter = total - floor(total)  (零头留给下次)

  ⑥ 余额更新
     balanceAfter = balanceBefore + amount

  ---
  四、数据流向（以具体数字为例）

  假设：沙箱运行 305.5 秒，creditsPerUnit = 1000，上一条 ledger 的 carryAfter = 0.2

  ┌────────────┬─────────────────────────────────────┬────────────────────┬──────────────────────────────────────────────┐
  │    步骤    │                计算                 │        结果        │                    写入表                    │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ① 时长     │ ceil(305500ms / 1000)               │ 306 秒             │ sandbox_run.runningSeconds                   │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ② 费用     │ 306 × 0.000046                      │ $0.014076          │ sandbox_run.costUsd + mira_usage.costSettled │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ③ 原始积分 │ 0.014076 × 1000                     │ 14.076             │ mira_credit.rawCreditAmount                  │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ④ 消息汇总 │ 14.076(sandbox) + 33.447(LLM)       │ sumRaw = 47.523    │ credit_ledger.sumRaw                         │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ⑤ 取整     │ floor(47.523 + 0.2) = floor(47.723) │ amount = -47       │ credit_ledger.amount                         │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ⑥ 零头     │ 47.723 - 47                         │ carryAfter = 0.723 │ credit_ledger.carryAfter                     │
  ├────────────┼─────────────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤
  │ ⑦ 余额     │ 500 + (-47)                         │ balanceAfter = 453 │ credit_ledger.balanceAfter                   │
  └────────────┴─────────────────────────────────────┴────────────────────┴──────────────────────────────────────────────┘

  六、排查要点

  ┌───────────────────────┬───────────────────────────────────────────────────────────────────────────────┐
  │       排查场景        │                                    查什么                                     │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ 沙箱用了但没扣费      │ sandbox_run 中 pauseAt IS NULL（没结算）或 usageId 为空（usage 写入失败）     │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ 重复扣费              │ mira_usage 中同一 vendorRequestId 是否有多条（应该被 UNIQUE 拦截）            │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ 积分扣多了            │ 检查 credit_ledger.sumRaw — 它是 LLM + tool + sandbox 的总和，不只是沙箱      │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ 积分扣少了            │ 检查 mira_credit 是否有对应 usageId 的记录；检查 credit_rate 是否有生效的汇率 │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ 余额不对              │ 查 credit_ledger 最近几条，验证 carryBefore → carryAfter 的连续性             │
  ├───────────────────────┼───────────────────────────────────────────────────────────────────────────────┤
  │ sandbox_run 永远 open │ pauseAndSettleSandbox 没被调用（消息异常退出时 fire-and-forget 可能丢失）     │
  └───────────────────────┴───────────────────────────────────────────────────────────────────────────────┘
