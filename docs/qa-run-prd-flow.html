/qa-run-prd [prd-path] [--source <dir>]

CMD 命令层 SONNET 生成/修复
P0 初始化
CMD 读取配置
输入
.env, CLAUDE.md
输出
projectContext (地址、凭证、技术栈)
CMD 初始化工作区
phase-0-init.md 2a-2f
i18n + Auth 基础设施
P1 读取 PRD + 变更检测
CMD 读取 PRD + 模块拆分
输入
PRD 文件（参数指定或默认 docs/prd/）
输出
按 ## 标题拆分的模块列表
CMD PRD Hash 变更检测
现有 *-prd.md 中 PRD-hash vs 当前模块 sha256
CMD 模块变更状态（每个模块独立判定）
none
哈希匹配
跳过生成
new
无现有 .md
从头生成
updated
哈希不匹配
增量更新
removed
模块已删除
test.describe.skip
标记废弃
P2 并行生成（仅 new / updated 模块）
SONNET e2e-orchestrator × N（每模块一个，并行）
输入
source:"prd", prdFiles, prdModuleScope, prdChangeMode, projectContext
输出
{slug}-prd.md, handoff.json, *.xlsx
{slug}.page.ts, {slug}-prd.test.ts
CMD POM 合并
多 orchestrator 生成
同页面的 POM 合并
CMD 验证门 V1-V5
逐一校验每个 orchestrator 结果
失败 → 流水线停止
CMD Excel 导出
excel-export-gate.md
CMD 结果检查
A: 无新 spec + 无删除
PRD 未变
结束
B: 无新 spec + 有删除模块
验证剩余测试
仍能通过
C: 有新 spec
继续修复流程
P3 交给 qa-fix-tests
CMD 执行 /qa-fix-tests --skip-baseline
输入
allGeneratedSpecs
CDP 探查真实页面 → 修复定位器/断言 → 回归验证
流水线结束
