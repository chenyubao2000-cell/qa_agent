## Method 1: Equivalence Partitioning

### Email Input Equivalence Classes

| Input Condition | Valid Equivalence Class | ID | Invalid Equivalence Class | ID |
|----------------|------------------------|-----|--------------------------|-----|
| Email format | Valid email (e.g., user@example.com) | V1 | Missing @ (e.g., "notanemail") | I1 |
| | | | Empty string | I2 |
| | | | Missing domain (e.g., "user@") | I3 |

**TC-CDP-SIGNIN-001**: 输入有效邮箱后继续按钮可用并进入密码步骤
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 用户在 /sign-in 页面
- **操作步骤:** 1. 在邮箱输入框中输入有效邮箱 "test@example.com" 2. 点击"继续"按钮
- **预期结果:** 页面进入密码输入步骤，显示"输入密码"标题
- **测试数据:** test@example.com (V1)

**TC-CDP-SIGNIN-002**: 输入无效邮箱格式后显示验证错误
- **优先级:** P1
- **测试类型:** 等价类划分
- **前置条件:** 用户在 /sign-in 页面
- **操作步骤:** 1. 在邮箱输入框中输入无效邮箱 "invalid-email" 2. 点击"继续"按钮
- **预期结果:** 显示错误信息"请输入有效的邮箱地址"
- **测试数据:** invalid-email (I1)

## Method 3: Cause-Effect Graph / Decision Table

| # | Email 非空 | Email 格式有效 | Password 非空 | Result |
|---|:---:|:---:|:---:|--------|
| 1 | N | - | - | 继续按钮禁用（Step 1） |
| 2 | Y | N | - | 显示邮箱格式错误（Step 1） |
| 3 | Y | Y | N | 继续按钮禁用（Step 2） |
| 4 | Y | Y | Y (wrong) | 显示凭证错误（Step 2） |
| 5 | Y | Y | Y (correct) | 登录成功，跳转 /task |

Rows 1-2 covered by TC-001~003. Rows 3-5 produce:

**TC-CDP-SIGNIN-005**: 输入正确密码后登录成功跳转到 /task
- **优先级:** P0
- **测试类型:** 因果图
- **前置条件:** 用户在 /sign-in 密码步骤（已输入有效邮箱并点击继续）
- **操作步骤:** 1. 在密码输入框中输入正确密码 2. 点击"继续"按钮
- **预期结果:** 页面跳转到 /task

**TC-CDP-SIGNIN-006**: 输入错误密码后显示凭证错误信息
- **优先级:** P1
- **测试类型:** 因果图
- **前置条件:** 用户在 /sign-in 密码步骤（已输入有效邮箱并点击继续）
- **操作步骤:** 1. 在密码输入框中输入错误密码 "wrongpassword" 2. 点击"继续"按钮

- **测试数据:** wrongpassword

## Method 4: State Transition Testing

State flow: S0 (Email) → S2 (Password) → S5 (Success) / S4 (Error)
              ↘ S1 (Email Error) → S0
S2 → S3 (Show Password) → S2 (Hide Password)
S2 → S0 (Edit / Back to Login)

**TC-CDP-SIGNIN-007**: 密码步骤点击"编辑"返回邮箱步骤
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 用户在 /sign-in 密码步骤
- **操作步骤:** 1. 点击"编辑"按钮
- **预期结果:** 页面返回邮箱输入步骤，显示"登录或注册"标题

**TC-CDP-SIGNIN-008**: 密码步骤点击"返回登录"返回邮箱步骤
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 用户在 /sign-in 密码步骤
- **操作步骤:** 1. 点击"返回登录"按钮
- **预期结果:** 页面返回邮箱输入步骤，显示"登录或注册"标题

**TC-CDP-SIGNIN-009**: 点击显示密码切换为明文显示
- **优先级:** P2
- **测试类型:** 状态迁移
- **前置条件:** 用户在 /sign-in 密码步骤，已输入密码
- **操作步骤:** 1. 点击"显示密码"按钮

**TC-CDP-SIGNIN-010**: 点击隐藏密码切换回密文显示
- **优先级:** P2
- **测试类型:** 状态迁移
- **前置条件:** 用户在 /sign-in 密码步骤，密码已为明文显示
- **操作步骤:** 1. 点击"隐藏密码"按钮
- **预期结果:** 密码输入框类型变为 password（密文），按钮文字变为"显示密码"

## Method 5: Scenario Method

**TC-CDP-SIGNIN-011**: 完整登录流程 — 邮箱 → 密码 → 成功跳转
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 用户在 /sign-in 页面
- **操作步骤:** 1. 确认页面标题为"登录或注册" 2. 确认 Google 和 Microsoft OAuth 按钮可见 3. 在邮箱输入框中输入有效邮箱 4. 点击"继续"按钮 5. 确认进入密码步骤 6. 在密码输入框中输入正确密码 7. 点击"继续"按钮
- **预期结果:** 页面跳转到 /task
