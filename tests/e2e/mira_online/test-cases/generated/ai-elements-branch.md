## Method 1: Equivalence Partitioning

**等价类**: AI 对话中的不同元素类型

| 等价类 | 有效/无效 | 元素 |
|--------|-----------|------|
| 用户消息 | 有效 | .is-user.group 容器 |
| 助手消息 | 有效 | .is-assistant.group 容器，含 Mira 标签 |
| 代码块 (CodeBlock) | 有效 | [data-language] 容器，含语法高亮 pre + 复制按钮 |
| 工具卡片 (Tool) | 有效 | Collapsible 容器，含工具名 + 状态 Badge |
| 无 AI 内容 | 无效 | 新任务页（无对话历史） |

**TC-BR-AIE-001**: 已完成任务页面显示用户消息和助手消息
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 用户已登录，导航至一个已完成的任务页面
- **操作步骤:** 1. 打开已有 AI 对话的任务页面 2. 检查对话日志区域
- **预期结果:** 页面包含 role="log" 的对话日志区域，至少存在一条用户消息（.is-user.group）和一条助手消息（.is-assistant.group），助手消息包含 "Mira" 标签

**TC-BR-AIE-002**: 助手消息中的代码块正确渲染（含 data-language 属性和语法高亮）
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 用户已登录，导航至含代码输出的已完成任务页面
- **操作步骤:** 1. 打开含代码块的任务页面 2. 定位 [data-language] 元素 3. 检查代码块结构
- **预期结果:** 代码块容器有 data-language 属性，内部有 pre > code 结构，header 区域显示语言标识，存在复制按钮（ghost variant button）

## Method 2: Boundary Value Analysis

**TC-BR-AIE-003**: 代码块复制按钮点击后图标切换为 CheckIcon
- **优先级:** P1
- **测试类型:** 边界值分析
- **前置条件:** 用户已登录，导航至含代码块的已完成任务
- **操作步骤:** 1. 定位代码块中的复制按钮 2. 点击复制按钮 3. 观察图标变化
- **预期结果:** 点击后按钮图标从 CopyIcon 切换为 CheckIcon（约 2 秒后恢复）

## Method 3: Cause-Effect Graph / Decision Table

**工具卡片 → 工作区面板的因果关系**

| 条件 | 动作 | 结果 |
|------|------|------|
| 点击工具卡片 | click | 打开工作区面板 |
| 工作区面板已打开 → 点击关闭 | click close | 关闭面板 |

**TC-BR-AIE-004**: 点击工具卡片打开工作区面板并显示代码查看器
- **优先级:** P0
- **测试类型:** 因果图
- **前置条件:** 用户已登录，导航至含工具调用的已完成任务页面
- **操作步骤:** 1. 定位工具卡片（.not-prose.rounded-md.border.cursor-pointer） 2. 点击工具卡片 3. 等待工作区面板出现
- **预期结果:** 工作区面板可见，标题为"Mira 的工作区"，面板内包含代码查看器（pre.shiki），显示时间线滑块（role=slider）

## Method 4: State Transition Testing

**状态转换**: S0（对话页面） ↔ S1（工作区面板打开）

**TC-BR-AIE-005**: 工作区面板打开后关闭恢复到对话视图
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 用户已登录，导航至含工具调用的已完成任务页面
- **操作步骤:** 1. 点击工具卡片打开工作区面板 2. 验证面板可见 3. 点击关闭按钮 4. 验证面板隐藏
- **预期结果:** 工作区面板从可见变为隐藏，对话区域恢复全宽显示

N/A — 工具运行中状态（input-streaming, input-available 等）需要实时任务执行，在已完成任务中不可观测。

## Method 5: Scenario Method

**TC-BR-AIE-006**: 完整的任务对话浏览场景——查看消息、代码块、工具卡片、完成状态
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 用户已登录，导航至一个含代码执行工具调用的已完成任务页面
- **操作步骤:** 1. 打开任务页面 2. 验证对话日志可见 3. 验证用户消息存在 4. 验证助手消息含 Mira 标签 5. 验证工具卡片显示工具名称（如"执行代码"） 6. 验证任务完成指示器（"任务已完成"） 7. 点击工具卡片 8. 验证工作区面板打开
- **预期结果:** 所有 AI 对话元素正确渲染，工具卡片交互正常

## Method 6: Error Guessing

**TC-BR-AIE-007**: 工具卡片工具名和描述文本正确显示
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 用户已登录，导航至含工具调用的已完成任务页面
- **操作步骤:** 1. 定位工具卡片 2. 检查工具名称文本 3. 检查工具描述文本
- **预期结果:** 工具卡片显示工具名称（含 .shrink-0.font-medium 样式的文本，如"执行代码"），且显示工具描述（含 .text-muted-foreground 样式的描述文本）

**TC-BR-AIE-008**: 工作区面板时间线滑块可见且可交互
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 用户已登录，工作区面板已打开
- **操作步骤:** 1. 点击工具卡片打开工作区面板 2. 定位时间线滑块（role=slider） 3. 验证滑块可见
- **预期结果:** 时间线滑块（input[role=slider]）可见，附近显示"实时"标签
