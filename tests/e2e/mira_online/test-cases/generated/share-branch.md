## Method 1: Equivalence Partitioning

将分享页面的输入（URL + token）划分为等价类：

| 等价类 | 输入条件 | 预期行为 |
|-------|---------|---------|
| 有效分享链接（有效 token + 有效 task ID） | URL 含有效 token | 显示任务内容（只读模式） |
| 无效 token 链接 | URL 无 token 或 token 无效 | 显示"链接无效"错误页 |
| 不存在的任务 | URL 含有效 token 但 task ID 不存在 | 显示"会话不存在"错误页 |

**TC-BR-SHARE-001**: 有效分享链接正常展示任务内容
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 存在一个已分享的任务，拥有有效的 shareToken
- **操作步骤:** 1. 使用有效 token 访问分享页面 URL
- **预期结果:** 页面正常加载，显示 Mira logo 链接、任务标题、对话记录区域（role="log"）

**TC-BR-SHARE-002**: 无效 token 显示链接无效错误
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 无
- **操作步骤:** 1. 访问分享页面 URL 但不携带 token 参数
- **预期结果:** 显示"链接无效"标题和"抱歉，此分享链接无效或已过期。"描述文案，以及"返回首页"链接

**TC-BR-SHARE-003**: 不存在的任务显示会话不存在错误
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 无
- **操作步骤:** 1. 使用有效 token 但不存在的 task ID 访问分享页面
- **预期结果:** 显示"会话不存在"标题和"抱歉，您访问的会话不存在或已被删除。"描述文案，以及"返回首页"链接

## Method 4: State Transition Testing

三个状态间无运行时转换（页面加载时由服务端决定渲染哪个状态）。错误页面的"返回首页"链接是唯一的导航交互：

**TC-BR-SHARE-004**: 链接无效页面点击返回首页导航到首页
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 已进入"链接无效"错误页
- **操作步骤:** 1. 点击"返回首页"链接
- **预期结果:** 页面导航到首页（URL 包含 "/"）

**TC-BR-SHARE-005**: 会话不存在页面点击返回首页导航到首页
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 已进入"会话不存在"错误页
- **操作步骤:** 1. 点击"返回首页"链接
- **预期结果:** 页面导航到首页（URL 包含 "/"）

## Method 5: Scenario Method

**TC-BR-SHARE-006**: 有效分享页面只读模式验证（无输入区域、无侧边栏）
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** 存在一个已分享的任务，拥有有效的 shareToken
- **操作步骤:** 1. 使用有效 token 访问分享页面 2. 检查页面布局元素
- **预期结果:** 页面不显示 textarea 输入框、不显示侧边栏切换按钮，标题以纯文本展示（无重命名按钮）

## Method 6: Error Guessing

**TC-BR-SHARE-007**: Mira logo 链接指向首页
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 存在一个已分享的任务，拥有有效的 shareToken
- **操作步骤:** 1. 使用有效 token 访问分享页面 2. 检查 Mira logo 链接的 href 属性
- **预期结果:** Mira logo 链接的 href 为 "/"
