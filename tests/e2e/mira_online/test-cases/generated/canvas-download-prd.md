## Method 1: Equivalence Partitioning

### 输入域划分

**文件格式等价类（下载格式选项数量）:**

| 等价类 | 原格式 | 可转换格式 | 示例 |
|--------|--------|-----------|------|
| 多格式可选（doc/docx） | docx | PDF、MD | docx 文件 |
| 多格式可选（pdf） | pdf | docx | PDF 文件 |
| 双格式可选（xlsx/xls） | xlsx | csv | xlsx 文件 |
| 双格式可选（csv） | csv | xlsx | csv 文件 |
| 双格式可选（ppt/pptx） | pptx | PDF | pptx 文件 |
| 多格式可选（md/txt） | md | docx、PDF | md 文件 |
| 仅原格式（图片） | png | — | png 文件 |
| 仅 xlsx（People Data） | — | xlsx | people-data 文件 |

**TC-PRD-CDLD-001**:docx 文件下载按钮弹出三个格式选项（原格式 / PDF / MD）
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 已登录；进入包含 docx 文件的任务页面；Canvas 已打开该 docx 文件
- **操作步骤:** 1. 进入任务页面 `/task/:taskId` 2. 点击 docx 文件卡片，Canvas 打开文件 3. 点击 Canvas 头部下载按钮
- **预期结果:** 弹出格式选择下拉框，仅展示「原格式（.docx）」「PDF」「MD」三个选项，无其他无效选项

**TC-PRD-CDLD-002**:png 图片文件下载按钮仅展示原格式选项
- **优先级:** P1
- **测试类型:** 等价类划分
- **前置条件:** 已登录；进入包含 png 图片的任务页面；Canvas 已打开该图片
- **操作步骤:** 1. 进入任务页面 2. 点击 png 文件卡片 3. 点击 Canvas 头部下载按钮
- **预期结果:** 格式下拉框仅展示「原格式（.png）」一个选项

**TC-PRD-CDLD-003**:People Data 文件下载仅提供 xlsx 格式
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 已登录；进入包含 People Data 文件的任务页面；Canvas 已打开该文件
- **操作步骤:** 1. 进入任务页面 2. 点击 People Data 文件卡片 3. 点击下载按钮
- **预期结果:** 直接下载 xlsx 文件，不弹出格式选择（或下拉框仅显示 xlsx 一个选项）

**TC-PRD-CDLD-004**:pdf 文件下载按钮弹出两个格式选项（原格式 / docx）
- **优先级:** P1
- **测试类型:** 等价类划分

## Method 2: Boundary Value Analysis

### 边界场景

**下载状态边界（下载中 / 空闲）:**

**TC-PRD-CDLD-005**:下载进行中时下载按钮处于加载态、不可重复点击
- **优先级:** P1
- **测试类型:** 边界值分析
- **前置条件:** 已登录；Canvas 已打开可转换格式文件（如 docx）；网络正常
- **操作步骤:** 1. 进入任务页面，打开 docx 文件 2. 点击下载按钮，选择 PDF 格式 3. 在下载进行中立即再次尝试点击下载按钮
- **预期结果:** 下载进行中按钮显示加载 spinner（Loader2Icon）；按钮处于 disabled 状态，无法重复触发

**TC-PRD-CDLD-006**:下载成功后按钮短暂显示成功状态（✓），随后恢复空闲
- **优先级:** P2
- **测试类型:** 边界值分析
- **前置条件:** 已登录；Canvas 已打开可下载的原格式文件（如 txt）
- **操作步骤:** 1. 进入任务页面，打开 txt 文件 2. 点击下载按钮，选择原格式下载 3. 等待下载完成
- **预期结果:** 下载成功后，按钮图标短暂变为绿色 CheckIcon，约 2 秒后恢复为 DownloadIcon

## Method 4: State Transition Testing

### 下载按钮状态机

```
idle → [点击下载，选格式] → generating（转换中）
                                ↓ 转换成功
                            downloading（下载中）
                                ↓ 下载成功
                            success（✓，2s）→ idle
                        generating → [转换失败] → error → idle（2s）
```

**TC-PRD-CDLD-010**:下载按钮完整状态流转：idle → generating → downloading → success → idle
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 已登录；Canvas 已打开 docx 文件；网络正常
- **操作步骤:** 1. 确认按钮初始为 idle（DownloadIcon） 2. 点击下载按钮，选择 PDF 格式 3. 观察按钮变为 generating（spinner） 4. 等待转换完成，按钮变为 downloading 5. 等待下载结束，按钮显示 success（✓） 6. 约 2 秒后按钮恢复 idle
- **预期结果:** 每个状态均可被观察到；最终按钮恢复为 DownloadIcon 空闲状态

**TC-PRD-CDLD-011**:任务区文件卡片下载按钮在下载结束后恢复可用
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 已登录；任务页面包含可下载文件卡片
- **操作步骤:** 1. 在任务结果区找到文件卡片 2. 点击文件卡片上的下载按钮 3. 等待下载完成 4. 再次点击下载按钮
- **预期结果:** 第一次下载完成后按钮恢复正常；第二次点击可成功再次触发下载，支持多次重复下载

## Method 5: Scenario Method

### 场景：Canvas 内完整下载流程

**TC-PRD-CDLD-012**:用户在 Canvas 内完成 xlsx 文件原格式下载端到端流程
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 已登录；进入包含 xlsx 文件的任务页面（文件有效，元数据完整）
- **操作步骤:** 1. 进入任务页面 `/task/:taskId` 2. 找到 xlsx 文件卡片，点击进入 Canvas 预览 3. 确认 Canvas 头部显示文件名和下载按钮 4. 点击下载按钮 5. 选择「原格式（.xlsx）」 6. 确认浏览器触发下载 7. 确认 toast 提示成功
- **预期结果:** xlsx 文件成功下载至本地；toast 提示「下载成功」；按钮恢复可用；支持再次下载

**TC-PRD-CDLD-013**:用户在任务结果区文件卡片直接下载文件
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 已登录；任务已生成包含文件的结果区域（至少一个文件卡片）
- **操作步骤:** 1. 进入已完成任务的详情页 2. 在结果区找到文件卡片（ArtifactEntry） 3. 点击文件卡片上的下载图标 4. 等待下载完成
- **预期结果:** 文件触发浏览器下载；下载期间按钮显示 spinner；成功后显示 ✓ 图标并 toast 提示「下载成功」

**TC-PRD-CDLD-014**:md 文件转换为 PDF 格式下载
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** 已登录；Canvas 已打开 md 文件（元数据完整）
- **操作步骤:** 1. 进入任务页面，打开 md 文件 2. 点击 Canvas 头部下载按钮 3. 选择「PDF」格式 4. 等待转换和下载完成
- **预期结果:** 按钮先显示转换 spinner；转换完成后触发 PDF 文件下载；toast 提示「下载成功」

**TC-PRD-CDLD-015**:csv 文件转换为 xlsx 格式下载
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** 已登录；Canvas 已打开 csv 文件（元数据完整）
- **操作步骤:** 1. 进入任务页面，打开 csv 文件 2. 点击 Canvas 头部下载按钮 3. 选择「xlsx」格式 4. 等待转换和下载完成
- **预期结果:** 触发 xlsx 文件下载；toast 提示成功

## Method 6: Error Guessing

**TC-PRD-CDLD-016**:文件加载失败时不展示下载入口
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 已登录；模拟文件加载失败（Canvas 错误状态）
- **操作步骤:** 1. 进入任务页面 2. 模拟文件获取失败（无效 r2Key 或网络中断） 3. 观察 Canvas 头部
- **预期结果:** Canvas 显示加载失败错误提示；下载按钮不可见或被禁用；不触发任何下载

**TC-PRD-CDLD-017**:文件元数据不完整时下载按钮禁用
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 已登录；文件缺少 r2Key 或 taskId（metadata 不完整）
- **操作步骤:** 1. 渲染缺少 metadata 的文件卡片 2. 尝试点击下载按钮
- **预期结果:** 下载按钮禁用（disabled）或点击后 toast 提示「文件信息不完整，无法转换」；不触发下载

**TC-PRD-CDLD-018**:多次快速点击下载按钮不触发多次下载请求
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 已登录；Canvas 已打开可下载文件
- **操作步骤:** 1. 进入任务页面，打开可下载文件 2. 快速连续三次点击下载按钮 3. 观察网络请求数量
- **预期结果:** 按钮在第一次点击后进入 disabled/loading 状态；后续点击被忽略；只发出一次下载请求

**TC-PRD-CDLD-019**:Canvas 头部下载按钮 hover 显示 tooltip「下载文件」
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 已登录；Canvas 已打开文件
- **操作步骤:** 1. 进入任务页面，打开文件 2. hover Canvas 头部下载按钮
- **预期结果:** 显示 tooltip 文字「Download file」（EN）/「下载文件」（ZH）
