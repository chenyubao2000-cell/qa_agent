# 测试用例：上传文件 input 展示样式（PRD 3.2.4）

## 功能说明

用户在 input 框中上传文件后，展示"文件胶囊/卡片"样式，包含：文件图标、文件名称、文件类型标签、文件大小；支持点击移除按钮取消单个文件；多文件纵向/换行排列；支持上传中状态、上传失败状态以及各类异常处理。

---

## Method 1: Equivalence Partitioning

### 等价类划分

**维度一：文件类型（合法 vs 非法）**

| 输入条件 | 有效等价类 | ID | 无效等价类 | ID |
|----------|-----------|-----|-----------|-----|
| 文件扩展名 | .pdf / .docx / .xlsx / .png / .jpg / .txt / .md / .csv / .ppt / .pptx / .json / .gif / .webp / .doc / .xls | V1 | 不支持的格式（如 .exe / .zip） | I1 |
| MIME 类型 | 与文件扩展名匹配的合法 MIME | V2 | MIME 无法识别（浏览器报告空字符串） | I2 |

**维度二：文件大小**

| 输入条件 | 有效等价类 | ID | 无效等价类 | ID |
|----------|-----------|-----|-----------|-----|
| 单文件大小 | 0 < size ≤ 20MB | V3 | size > 20MB（超出单文件限制） | I3 |
| 文件大小可获取 | 文件 size 属性为正整数 | V4 | 文件 size 属性为 0 或无法获取 | I4 |

**维度三：文件数量**

| 输入条件 | 有效等价类 | ID | 无效等价类 | ID |
|----------|-----------|-----|-----------|-----|
| 文件数量 | 1 ≤ count ≤ 10 | V5 | count > 10（超出最大文件数） | I5 |
| 多文件 | 2 ≤ count ≤ 10 | V6 | — | — |

**用例设计：**

**TC-PRD-UPINP-001**：单个支持类型文件展示完整胶囊信息（覆盖 V1、V2、V3、V4、V5）
**TC-PRD-UPINP-002**：多个文件展示多条胶囊（覆盖 V5、V6）
**TC-PRD-UPINP-003**：不支持的文件类型触发错误提示，不进入成功展示态（覆盖 I1）
**TC-PRD-UPINP-004**：单文件超过 20MB 触发大小限制错误（覆盖 I3）
**TC-PRD-UPINP-005**：文件大小无法获取时展示"--"（覆盖 I4）
**TC-PRD-UPINP-006**：上传文件数超过 10 个触发数量限制错误（覆盖 I5）

## Method 2: Boundary Value Analysis

### 边界值分析

**边界：单文件大小限制（最大 20MB = 20 * 1024 * 1024 字节）**

| 边界 | 值 | 期望行为 |
|--------|-----|---------|
| 边界内 | 20MB - 1 byte | 展示胶囊，显示文件大小 |
| 恰好边界 | 20MB | 展示胶囊，显示文件大小（边界值允许） |
| 超出边界 | 20MB + 1 byte | 触发"文件超出大小限制"提示 |

**边界：文件数量限制（最大 10 个）**

| 边界 | 值 | 期望行为 |
|--------|-----|---------|
| 边界内 | 10 个文件 | 全部展示胶囊 |
| 超出边界 | 11 个文件 | 触发"超出最大文件数"提示 |

**边界：文件名长度（超长需省略）**

| 边界 | 值 | 期望行为 |
|--------|-----|---------|
| 正常 | ≤ 40 字符 | 完整展示 |
| 超长 | > 40 字符 | 文件名截断显示（末尾省略号） |

**TC-PRD-UPINP-007**：文件大小恰好为 20MB 时展示完整胶囊（边界值，允许上传）
**TC-PRD-UPINP-008**：文件名超长时胶囊截断展示（不溢出布局）

## Method 4: State Transition Testing

### 状态迁移测试

文件胶囊的状态机：

```
S0 未上传 → [选择文件] → S1 上传中（isUploading=true）
S1 上传中  → [上传成功] → S2 已上传（显示完整信息）
S1 上传中  → [上传失败] → S3 上传失败（红色提示）
S2 已上传  → [点击移除] → S0 未上传
S3 上传失败 → [点击移除] → S0 未上传
```

**TC-PRD-UPINP-011**：上传成功后胶囊从"上传中"状态迁移至展示完整信息状态
**TC-PRD-UPINP-012**：上传失败后胶囊展示失败状态（标红），支持移除操作
**TC-PRD-UPINP-013**：点击移除按钮后胶囊消失，上传列表同步更新

## Method 5: Scenario Method

### 场景法

**基本流：单文件上传并展示完整胶囊**
1. 用户在 input 框点击上传按钮
2. 选择一个支持类型的文件（如 sample.pdf）
3. 文件上传成功
4. input 区域出现文件胶囊，显示 PDF 图标、文件名、类型"PDF"、大小"xx.x KB/MB"
5. 移除按钮（X）可点击
6. 用户点击 X，胶囊消失

**备选流 1：多文件上传**
- 场景：依次上传 3 个文件（pdf + xlsx + png）
- 期望：3 个胶囊纵向/换行排列，不挤压文本输入区域

**备选流 2：图片文件上传**
- 场景：上传一张 .png 图片
- 期望：胶囊内显示图片缩略图（而非文件类型图标），类型标签为"Image"

**备选流 3：未知类型文件**
- 场景：上传扩展名极少见的合法文件（type 为 ""）
- 期望：类型标签显示"File"（通用兜底图标），不报错

**TC-PRD-UPINP-014**：完整单文件上传→展示→移除流程（主流程 E2E）
**TC-PRD-UPINP-015**：多文件上传后换行排列且不影响文本输入

## Method 6: Error Guessing

### 错误猜测

根据 PRD 3.2.4.3 质量与安全以及常见前端上传问题：

1. **文件类型无法识别**：MIME 类型为空字符串时，类型标签应显示"File"（通用图标），而非崩溃
2. **文件大小无法获取**：`size` 为 0 或 `undefined` 时，大小栏显示"--"，不显示"0 B"
3. **超大文件直接提示**：命中大小限制，直接提示并不进入成功展示态
4. **不支持类型文件**：直接提示，不展示胶囊
5. **XSS 文件名**：文件名含特殊字符（如 `<script>test.pdf`），应正常截断展示，不执行脚本
6. **上传中时点击 Submit**：仍有上传中的附件时，Submit 应等待上传完成再提交（不丢失文件）

**TC-PRD-UPINP-016**：MIME 类型为空时胶囊类型标签降级显示"File"
**TC-PRD-UPINP-017**：文件名包含 HTML 特殊字符时胶囊安全展示不执行脚本
