## Method 1: Equivalence Partitioning

**输入域分析**：Canvas 支持多种文件类型，每种类型对应不同预览器，每个预览器的交互集合不同。

**等价类划分**：

| 输入条件 | 有效等价类 | ID | 无效等价类 | ID |
|---------|-----------|-----|-----------|-----|
| 文件类型 | PDF（支持翻页+缩放） | V1 | 损坏文件（加载失败） | I1 |
| 文件类型 | PPT/PPTX（支持幻灯片切换+缩略图） | V2 | 已被删除的文件 | I2 |
| 文件类型 | 图片（支持缩放+重置） | V3 | 格式不支持预览（兜底页） | I3 |
| 文件类型 | 文本/Markdown/JSON（滚动阅读） | V4 | | |
| 文件类型 | Excel/CSV（表格预览） | V5 | | |
| 文件类型 | Word/DOCX（文档预览） | V6 | | |
| 全屏状态 | 非全屏（正常尺寸） | V7 | | |
| 全屏状态 | 全屏（最大化） | V8 | | |

**TC-PRD-CVPV-001**: Canvas 面板全屏按钮可最大化面板
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 已打开任务详情页，Canvas 面板中已加载 PDF 文件（非全屏状态）
- **操作步骤:** 1. 导航至包含 PDF 文件的任务 2. 点击文件卡片打开 Canvas 预览 3. 点击 Canvas 头部 Maximize（全屏）按钮
- **预期结果:** Canvas 面板切换为全屏模式；Maximize 按钮图标变为 Restore（还原）图标

**TC-PRD-CVPV-002**: Canvas 全屏状态下点击还原按钮恢复正常尺寸

- **前置条件:** Canvas 面板已处于全屏（Maximize）状态
- **操作步骤:** 1. 导航至包含文件的任务 2. 点击文件卡片 → 点击 Maximize 进入全屏 3. 点击 Restore（还原）按钮
- **预期结果:** Canvas 面板恢复正常尺寸；按钮图标变回 Maximize（全屏）图标

- **预期结果:** Canvas 面板关闭，回到任务主内容区域

**TC-PRD-CVPV-004**: PPT 文件在 Canvas 中显示幻灯片主视图与缩略图侧栏

## Method 4: State Transition Testing

**状态机**：Canvas 面板的状态转换。

```
[空状态/无文件] → 点击文件卡片 → [加载中] → 加载成功 → [预览中-正常尺寸]
                                              ↓ 加载失败 → [加载失败]
[预览中-正常尺寸] → 点击 Maximize → [预览中-全屏]
[预览中-全屏] → 点击 Restore → [预览中-正常尺寸]
[预览中-正常尺寸] → 点击 Close → [空状态]
[加载失败] → 点击 Retry → [加载中]
```

**TC-PRD-CVPV-015**: Canvas 从空状态打开文件后进入预览状态
- **优先级:** P0
- **测试类型:** 状态迁移
- **前置条件:** 任务详情页无打开的 Canvas 文件
- **操作步骤:** 1. 导航至包含文件的任务 2. 点击文件卡片
- **预期结果:** Canvas 面板从空状态（"Select a file to preview"）过渡到文件预览状态，显示文件名和文件内容

**TC-PRD-CVPV-016**: Canvas 预览中点击 Close 返回空状态
- **优先级:** P0
- **测试类型:** 状态迁移
- **前置条件:** Canvas 面板正在预览某文件
- **操作步骤:** 1. 打开 Canvas 预览 2. 点击 Close 按钮
- **预期结果:** Canvas 面板关闭（空状态或隐藏），文件预览内容消失

**TC-PRD-CVPV-017**: Canvas 预览 → 全屏 → 还原完整状态迁移
- **优先级:** P0
- **测试类型:** 状态迁移
- **前置条件:** Canvas 面板正在预览文件，处于正常尺寸状态
- **操作步骤:** 1. 点击 Maximize 按钮 2. 验证全屏状态 3. 点击 Restore 按钮 4. 验证恢复正常尺寸
- **预期结果:** 状态机完整迁移：正常 → 全屏 → 正常；每步按钮图标和 tooltip 均正确切换

**TC-PRD-CVPV-018**: 加载失败后点击 Retry 重新触发加载
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** Canvas 加载文件失败，显示错误提示和 Retry 按钮
- **操作步骤:** 1. 触发文件加载失败 2. 点击 Retry（立即刷新）按钮
- **预期结果:** Canvas 重新进入加载状态（显示加载动画），尝试重新获取文件内容

## Method 5: Scenario Method

**基本流**：用户打开任务 → 点击文件卡片 → Canvas 加载文件 → 对文件进行预览操作（翻页/全屏）→ 关闭 Canvas。

**替代流**：
- A1：PPT 文件 → 通过缩略图切换幻灯片
- A2：PPT 文件 → 通过鼠标滚轮翻页
- A3：PDF 文件 → 通过工具栏翻页 + 缩放
- A4：文件加载失败 → 点击 Retry
- A5：任意文件 → 全屏查看 → 还原

**TC-PRD-CVPV-019**: PPT 完整预览流程：打开 → 缩略图导航 → 翻页
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 任务中存在多页 .pptx 文件
- **操作步骤:** 1. 导航至包含 pptx 文件的任务 2. 点击 pptx 文件卡片打开 Canvas 3. 等待 PPT 渲染完成 4. 在左侧缩略图中点击第2张幻灯片 5. 验证主视图切换到第2张
- **预期结果:** 点击缩略图后主视图切换到对应幻灯片；页码指示器更新为 "2 / N"；被选中的缩略图边框高亮

**TC-PRD-CVPV-020**: PPT 预览通过上/下翻页按钮逐张切换
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** Canvas 中已加载多页 PPT，当前第1张
- **操作步骤:** 1. 打开 pptx 文件 2. 等待渲染 3. 通过键盘右箭头或下一张按钮导航到第2张 4. 再导航到第3张
- **预期结果:** 每次翻页主视图更新，页码指示器显示 "当前页 / 总页数"，缩略图中对应项高亮

**TC-PRD-CVPV-021**: PDF 完整预览流程：打开 → 翻页 → 缩放
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 任务中存在多页 PDF 文件
- **操作步骤:** 1. 点击 PDF 文件卡片打开 Canvas 2. 等待 PDF 渲染 3. 点击右翻页按钮到第2页 4. 点击 ZoomIn 按钮放大 5. 点击 ZoomOut 按钮缩小
- **预期结果:** 翻页后页码更新为 "2 / N"；缩放百分比随操作更新（如 125%，100%）；内容清晰可见

**TC-PRD-CVPV-022**: 全屏查看文件后还原完整用户旅程
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 任务中存在 PDF 或图片文件
- **操作步骤:** 1. 点击文件卡片打开 Canvas 2. 点击 Maximize 进入全屏 3. 在全屏中查看文件内容 4. 点击 Restore 还原
- **预期结果:** 全屏模式下文件内容占据更大区域；还原后布局恢复正常；文件内容仍然可见

**TC-PRD-CVPV-023**: 不支持格式文件完整流程：打开 → 兜底页 → 下载
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** 任务中存在不支持预览的格式文件
- **操作步骤:** 1. 点击不支持格式的文件卡片 2. 观察 Canvas 内容 3. 点击 Canvas 兜底页中的下载按钮
- **预期结果:** Canvas 显示格式不支持的兜底页，包含说明文字和下载按钮；点击下载按钮触发文件下载

## Method 6: Error Guessing

**错误猜测**：基于 Canvas 的渲染特性和已知风险点。

**TC-PRD-CVPV-024**: PPT 单页文件切换页面时不出错
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 任务中存在只有1张幻灯片的 pptx 文件
- **操作步骤:** 1. 点击单页 pptx 文件卡片 2. 观察翻页按钮状态和页码
- **预期结果:** 页码显示 "1 / 1"；上翻/下翻按钮均禁用或不可操作；不出现 JS 错误

**TC-PRD-CVPV-025**: 快速连续点击 Maximize/Restore 不出现布局错乱
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** Canvas 面板正在预览文件
- **操作步骤:** 1. 快速连续点击 Maximize 和 Restore 按钮（3次以上）
- **预期结果:** 最终状态与最后一次操作一致；无布局错乱、白屏或渲染异常

**TC-PRD-CVPV-026**: 切换不同类型文件时 Canvas 内容正确更新
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 任务中同时存在 PDF 和 PPT 文件
- **操作步骤:** 1. 点击 PDF 文件卡片 → 等待加载 2. 点击 PPT 文件卡片
- **预期结果:** Canvas 切换为 PPT 预览器（含缩略图侧栏）；不保留 PDF 的页码状态；无残留 PDF 渲染内容

**TC-PRD-CVPV-027**: 文件名超长时 Canvas 头部文件名不破坏布局
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 任务中存在文件名很长（>50字符）的文件
- **操作步骤:** 1. 点击超长文件名的文件卡片 2. 观察 Canvas 头部文件名显示
- **预期结果:** 文件名以截断（truncate + title属性）方式显示，不挤压右侧操作按钮区域；hover 可见完整文件名

**TC-PRD-CVPV-028**: PDF 缩放到最小值（0.5x）后 ZoomOut 按钮禁用
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** Canvas 中已加载 PDF 文件
- **操作步骤:** 1. 打开 PDF 文件 2. 多次点击 ZoomOut 直到缩放到最小
- **预期结果:** 缩放比例停留在 50%（最小值）；ZoomOut 按钮禁用；不出现负数或0%缩放

**TC-PRD-CVPV-029**: PDF 缩放到最大值（3x）后 ZoomIn 按钮禁用
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** Canvas 中已加载 PDF 文件
- **操作步骤:** 1. 打开 PDF 文件 2. 多次点击 ZoomIn 直到缩放到最大
- **预期结果:** 缩放比例停留在 300%（最大值）；ZoomIn 按钮禁用
