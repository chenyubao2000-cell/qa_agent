# MIRA-1318: 场景发送后应导航到新对话页 — Verify-Fix 测试用例

> **Bug**: 点击发送后页面未导航到 `/task/{id}` 新对话页
> **Expected**: 点击场景建议卡片 → 填入输入框 → 点击发送 → 导航到 `/task/{taskId}`
> **Mode**: verify-fix（断言目标 = expectedBehavior，而非当前页面状态）

---

## Method 1: Equivalence Partitioning

**TC-VF-TSS-001**: 点击场景建议后填入内容，点击发送应导航到新对话页
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 已登录；位于 /task 页面；场景建议卡片可见
- **操作步骤:** 1. 打开 /task 页面 2. 等待场景建议卡片渲染 3. 点击第一个场景建议卡片 4. 确认输入框已填入场景文本 5. 确认发送按钮已启用 6. console.log 当前 URL（点击前） 7. 点击发送按钮 8. console.log 当前 URL（点击后）
- **预期结果:** 点击发送后页面 URL 变更为 /task/{taskId}（匹配 /task/[a-zA-Z0-9]+ 模式）；新对话页加载完成

**TC-VF-TSS-002**: 新对话页应显示已发送的场景消息内容
- **优先级:** P0
- **测试类型:** 等价类划分
- **前置条件:** 已登录；位于 /task 页面；场景建议卡片可见
- **操作步骤:** 1. 打开 /task 页面 2. 点击第一个场景建议卡片，记录填入的文本 3. 点击发送按钮 4. 等待导航到 /task/{taskId} 5. 等待对话日志加载
- **预期结果:** 导航到 /task/{taskId} 后，对话日志区域（role=log）可见；页面包含刚刚发送的场景文本内容

---

## Method 2: Boundary Value Analysis

**TC-VF-TSS-003**: 输入框为空时点击发送按钮，不应导航到新对话页
- **优先级:** P1
- **测试类型:** 边界值分析
- **前置条件:** 已登录；位于 /task 页面；未点击任何场景建议
- **操作步骤:** 1. 打开 /task 页面 2. 确认输入框为空 3. 检查发送按钮状态 4. 尝试点击发送按钮（如可点击） 5. 记录 URL 变化
- **预期结果:** 发送按钮处于 disabled 状态（无法点击）；或点击后页面 URL 保持 /task，不导航到 /task/{taskId}

## Method 5: Scenario Method

**TC-VF-TSS-004**: 场景建议卡片在 /task 页面可见且可点击（回归验证）
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** 已登录；位于 /task 页面
- **操作步骤:** 1. 打开 /task 页面 2. 等待页面加载完成 3. 验证"试试以下场景"标签文本可见 4. 验证场景建议卡片列表不为空（至少 1 个卡片可见） 5. 验证每个卡片均可点击（enabled 状态）
- **预期结果:** 页面上可见场景建议区域；至少有 1 个场景卡片按钮；卡片按钮处于可点击状态（非 disabled）
