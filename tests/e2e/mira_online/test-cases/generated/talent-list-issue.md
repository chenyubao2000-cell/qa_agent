# 人才列表网格布局 - 测试用例文档
<!-- issue: MIRA-1249 | feature-slug: talent-list | source: issue -->

## 功能描述

当人才搜索结果仅返回单人才时，最大化面板后，CSS container query 网格仍然根据容器宽度创建2-4列，导致75%的空白区域。预期行为：单张人才卡片应填满可用宽度或居中显示，不应出现大量空白占位列。

## 测试范围

- 分享页面（Share page）：`/share/C2J66q10Qw3Fw1uS?token=...`
- 功能区：人才搜索结果卡片 → 侧边面板 → 最大化模式
- Bug 重现：单人才时最大化面板出现空列
- 对比参照：6人才时最大化面板的正常布局

## Method 1: Equivalence Partitioning

等价类划分：按人才数量和视口宽度划分

**有效等价类：**
- EC1：人才数 = 1（单人才，触发Bug场景）
- EC2：人才数 = 6（多人才，对比正常场景）
- EC3：视口宽度 ≥ 1534px（4列断点）
- EC4：视口宽度 1158-1533px（3列断点）
- EC5：视口宽度 782-1157px（2列断点）
- EC6：视口宽度 < 782px（1列，正常）

## Method 6: Error Guessing

- **预期结果:** 最终面板状态与最后一次操作一致（最大化）；无布局错乱或双重面板出现
- **测试数据:** 无

- **优先级:** P2
- **测试类型:** 错误猜测
