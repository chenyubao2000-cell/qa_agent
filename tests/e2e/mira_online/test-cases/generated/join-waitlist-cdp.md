## Method 1: Equivalence Partitioning

### 等价类表

| 输入条件 | 有效等价类 | ID | 无效等价类 | ID |

| 工作邮箱 | 合法邮箱 name@company.com | V1 | 空值 | I1 |
| | | | 无 @ 符号 "notanemail" | I2 |
| 姓名 | 非空字符串 | V2 | 空值（若必填） | I3 |
| 公司 | 非空字符串 | V3 | — | — |
| 岗位/角色 | 非空字符串 | V4 | — | — |
| 使用场景 | 非空文本 | V5 | — | — |

**TC-CDP-JW-001**: 使用合法邮箱填写表单（覆盖 V1-V5）
- **优先级:** P1
- **测试类型:** 等价类划分
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 填写有效邮箱 2. 填写姓名 3. 填写公司 4. 填写角色 5. 填写使用场景
- **预期结果:** 所有字段正常接受输入，发送验证码按钮启用

**TC-CDP-JW-002**: 邮箱为空时提交
- **优先级:** P1
- **测试类型:** 等价类划分
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 不填写邮箱 2. 点击提交申请
- **预期结果:** 显示错误提示：请先验证邮箱后再提交
- **测试数据:** 邮箱字段留空

**TC-CDP-JW-003**: 无效邮箱格式
- **优先级:** P1
- **测试类型:** 等价类划分
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 在邮箱字段输入 "notanemail" 2. 观察发送验证码按钮状态
- **预期结果:** 发送验证码按钮保持禁用或提示邮箱格式错误
- **测试数据:** "notanemail"

## Method 4: State Transition Testing

### 状态流转图

```
S0 (空表单) --[填写邮箱]--> S2 (邮箱已填写，发送验证码启用)
S2 --[点击发送验证码]--> S3 (验证码已发送，按钮倒计时禁用)
S0 --[点击提交]--> S1 (toast: 请先验证邮箱)
S0 --[点击取消]--> 上一页 (history.back)
```

**TC-CDP-JW-005**: 填写邮箱后发送验证码按钮从禁用变为启用（S0 → S2）
- **优先级:** P0
- **测试类型:** 状态迁移
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 确认发送验证码按钮初始为禁用 2. 在邮箱字段输入有效邮箱 3. 观察发送验证码按钮状态
- **预期结果:** 发送验证码按钮变为启用状态
- **测试数据:** email: test@company.com

**TC-CDP-JW-006**: 点击发送验证码后按钮进入倒计时（S2 → S3）
- **优先级:** P1
- **测试类型:** 状态迁移
- **前置条件:** 导航到 /join-waitlist，填写有效邮箱
- **操作步骤:** 1. 在邮箱字段输入有效邮箱 2. 点击发送验证码按钮 3. 观察按钮状态
- **预期结果:** 发送验证码按钮变为禁用（倒计时状态）
- **测试数据:** email: test@company.com

## Method 5: Scenario Method

**TC-CDP-JW-007**: 完整申请流程 — 打开页面 → 确认表单元素 → 填写所有字段
- **优先级:** P0
- **测试类型:** 场景法
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 验证页面标题"加入等待名单"可见 2. 验证描述文字可见 3. 验证邮箱、姓名、公司、角色、使用场景字段可见 4. 验证提交申请和取消按钮可见
- **预期结果:** 所有表单元素正确渲染

**TC-CDP-JW-008**: 取消操作 — 点击取消按钮返回上一页
- **优先级:** P1
- **测试类型:** 场景法
- **前置条件:** 从首页导航到 /join-waitlist
- **操作步骤:** 1. 先导航到首页 2. 导航到 /join-waitlist 3. 点击取消按钮
- **预期结果:** 页面返回上一页（history.back()）

## Method 6: Error Guessing

**TC-CDP-JW-009**: 多次快速点击提交按钮
- **优先级:** P2
- **测试类型:** 错误猜测
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 不填写任何字段 2. 快速连续点击提交申请按钮两次
- **预期结果:** 只显示一次 toast 错误提示，不会出现重复提示或页面异常

**TC-CDP-JW-010**: 发送验证码按钮初始状态为禁用
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 导航到 /join-waitlist
- **操作步骤:** 1. 页面加载完成后 2. 检查发送验证码按钮状态
- **预期结果:** 发送验证码按钮为禁用状态（邮箱为空时不可发送）
