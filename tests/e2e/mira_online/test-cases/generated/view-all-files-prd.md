# 测试用例：查看所有文件（3.2.3）

**功能模块**:查看此任务中的所有文件
**来源**:PRD V0.2.8 Canvas-预览&下载 §3.2.3
**生成日期**:2026-03-29

## Method 1: Equivalence Partitioning

### 有效等价类

**EC-VF-01**:任务上下文中文件数 ≥ 1 → 展示"查看所有文件"入口按钮
**EC-VF-02**:任务上下文中文件数 = 0 → 不展示"查看所有文件"按钮
**EC-VF-03**:点击"查看所有文件"按钮 → 居中弹窗打开
**EC-VF-04**:弹窗内文件列表成功加载 → 按日期分组展示
**EC-VF-05**:点击文件卡片（非批量选择模式）→ 全屏预览打开
**EC-VF-06**:点击单个文件下载按钮 → 触发下载
**EC-VF-07**:点击批量下载图标 → 进入多选模式
**EC-VF-08**:批量选中文件后点击"批量下载" → 打包为 zip 下载
**EC-VF-09**:批量选择模式点击"取消" → 退出多选模式
**EC-VF-10**:批量选择模式点击"全选(N)" → 全部文件被选中

### 无效等价类

**EC-VF-11**:文件列表加载失败 → 展示错误提示与「立即刷新」按钮
**EC-VF-12**:批量下载进行中 → toast 显示「打包中...」
**EC-VF-13**:打包失败 → toast 显示「打包失败，请稍后重新尝试！」
**EC-VF-14**:单个文件下载失败 → 提示「文件下载失败，请检查网络后重试」

- **测试类型:** 等价类划分

- **操作步骤:** 1. 导航至包含文件的任务详情页  2. 观察页面顶部工具栏区域  3. 观察任务结果区域底部

- **优先级:** P1
- **测试类型:** 等价类划分

- **预期结果:** 页面顶部工具栏不显示查看所有文件图标按钮；任务结果区域不显示"查看此任务中所有文件"文字按钮

**TC-PRD-VF-003**:点击顶部工具栏入口打开弹窗
- **优先级:** P0
- **测试类型:** 等价类划分

- **预期结果:** 居中弹窗打开；弹窗标题显示"All files in this task"（或"此任务中的所有文件"）；文件列表按日期分组展示

**TC-PRD-VF-004**:点击任务结果区底部入口打开弹窗
- **优先级:** P0
- **测试类型:** 等价类划分

**TC-PRD-VF-005**:文件列表按日期分组展示

## Method 2: Boundary Value Analysis

**边界点**:
- 文件数 = 0（临界下边界）→ 不显示入口
- 文件数 = 1（临界值）→ 显示入口，弹窗内有 1 个文件卡片
- 批量选择：已选 0 个文件 → 批量下载按钮禁用
- 批量选择：已选 1 个文件 → 批量下载按钮启用
- 批量选择：全选所有文件 → 全选框选中状态

---

**TC-PRD-VF-007**:文件数恰好为 1 时显示入口
- **优先级:** P0
- **测试类型:** 边界值分析
- **前置条件:** 已登录；存在仅包含 1 个文件的任务
- **操作步骤:** 1. 导航至仅含 1 个文件的任务详情页  2. 观察页面顶部工具栏
- **预期结果:** 顶部工具栏显示"查看所有文件"图标按钮（触发条件 ≥1 满足）

**TC-PRD-VF-008**:批量模式未选文件时批量下载按钮禁用

- **测试类型:** 边界值分析

## Method 4: State Transition Testing

**状态定义**:
- S0：弹窗关闭
- S1：弹窗打开（正常列表模式）
- S2：弹窗打开（批量选择模式，无选中文件）
- S3：弹窗打开（批量选择模式，有选中文件）
- S4：全屏预览模式

**状态迁移**:
- S0 → S1：点击"查看所有文件"入口
- S1 → S0：点击弹窗关闭按钮
- S1 → S2：点击顶部"批量下载"图标
- S2 → S1：点击"取消"
- S2 → S3：选中至少 1 个文件
- S3 → S2：取消所有选中文件
- S3 → S0（下载后）：完成批量下载后退出选择模式，弹窗关闭
- S1 → S4：点击文件卡片进入全屏预览
- S4 → S1：关闭全屏预览

## Method 5: Scenario Method

### 场景一：用户任务完成后浏览所有文件并下载

1. Agent 完成任务，生成多个文件
2. 用户在顶部工具栏点击"查看所有文件"图标
3. 弹窗打开，显示按日期分组的文件列表
4. 用户逐一查看文件，点击文件卡片进入全屏预览
5. 用户在全屏预览中关闭预览，回到文件列表
6. 用户点击某文件的下载按钮，单独下载该文件
7. 关闭弹窗

### 场景二：用户批量下载所有文件

1. 用户打开"查看所有文件"弹窗
2. 用户点击批量下载图标进入多选模式
3. 用户点击"全选(N)"全选所有文件
4. 用户点击"批量下载"触发打包
5. 浏览器下载 zip 压缩包

### 场景三：任务执行中实时查看已生成文件

1. Agent 正在执行任务中（未完成）
2. 用户点击"查看所有文件"入口（仅展示已生成的文件）
3. 弹窗中展示已生成的部分文件
4. 用户关闭弹窗后等待任务完成
5. 再次打开弹窗查看完整文件列表

## Method 6: Error Guessing

**错误猜测点**:

1. **网络中断时点击批量下载** → 打包失败，toast 应提示「打包失败，请稍后重新尝试！」
2. **快速连续点击批量下载** → 防止重复触发，按钮在下载中应禁用
3. **文件列表为空时点击批量下载图标** → 图标按钮应禁用
4. **批量下载进行中关闭弹窗** → 行为确认（下载是否继续）
5. **Escape 键在全屏预览中关闭** → 键盘事件正确绑定
6. **弹窗在加载中时点击文件卡片** → 不应崩溃或产生异常

---

- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 已登录；弹窗已打开；文件列表加载完成但为空（0 个文件）
- **操作步骤:** 1. 打开"查看所有文件"弹窗  2. 等待文件列表加载（列表为空）  3. 观察顶部批量下载图标按钮的状态
- **预期结果:** 批量下载图标按钮处于禁用状态（disabled 属性）；不能点击进入多选模式

- **优先级:** P1

- **操作步骤:** 1. 在批量选择模式下选中文件  2. 点击"批量下载"触发打包  3. 在打包进行中观察"批量下载"按钮状态
- **预期结果:** 打包进行中按钮显示加载动画且处于禁用状态；防止重复点击

- **操作步骤:** 1. 全屏预览已打开  2. 按下键盘 Escape 键
- **预期结果:** 全屏预览关闭；返回文件列表弹窗（S1 状态）

**TC-PRD-VF-023**:批量下载打包失败显示 toast 提示
- **优先级:** P1
- **测试类型:** 错误猜测
- **前置条件:** 已登录；弹窗已打开；处于批量选择模式且已选中文件；网络不稳定或服务器错误
- **操作步骤:** 1. 进入批量选择模式并选中文件  2. 在网络故障条件下点击"批量下载"  3. 等待打包结果
- **预期结果:** toast 显示打包失败提示（"打包失败，请稍后重新尝试！"或"Packing failed, please try again later!"）；不继续下载
