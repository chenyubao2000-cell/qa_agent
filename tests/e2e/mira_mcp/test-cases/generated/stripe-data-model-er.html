4. 数据流走查（按 REQ）

🔵 蓝线 — 订阅生效（REQ-003 + REQ-004）
Stripe Checkout 支付成功
  ↓
checkout.session.completed webhook
  ↓ 同事务
1) INSERT stripe_webhook_event (event_id 幂等)
2) UPDATE user 的 7 个 stripe_* 字段
3) INSERT credit_ledger
   - txn_type = 'recharge'
   - amount = system_config["billing.tier.{tier}.credits"] × user.seats
   - idempotency_key = event_id (UK 防重复)
   - external_ref = event_id (备份)
🟢 绿线 — 月度续费 + Credit 重置（REQ-006）
Stripe 自动扣款成功 → invoice.paid webhook
  ↓
方案 A: credit_ledger 写两条
   - 1 条 expire (amount = -current_balance, 清零)
   - 1 条 recharge (amount = tier 满额)

方案 B（推荐）: credit_ledger 写一条 reset
   - amount = tier 满额（直接覆盖 balance_after）
   - 需 ALTER chk_credit_ledger_txn_type 加 'reset'

  + UPDATE user.current_period_end = next_billing_date
🔴 红线 — Credit 消费 / 耗尽（REQ-007，已存在不动）
用户发任务
  ↓
messages INSERT → usage 记录调用 → credit 算 raw_credit_amount
  ↓
credit_ledger INSERT (txn_type='consume', amount 为负)
  ↓
REQ-007 拦截：
  if balance_after < 任务成本 → 显示 credits_exhausted 弹窗
  resetDate 来源 = user.current_period_end
🟡 黄线 — 扣款失败 / 取消（REQ-008 / REQ-009）
invoice.payment_failed
  ↓
UPDATE user.subscription_status = 'past_due'
  + INSERT stripe_webhook_event
  （不动 credit_ledger，past_due 期间用户仍能用）

customer.subscription.deleted（周期末）
  ↓
UPDATE user.tier = 'free', subscription_status = 'canceled'
  + INSERT stripe_webhook_event
  （Credit 不动，让用户用到当前余额自然耗尽 = 周期末）
